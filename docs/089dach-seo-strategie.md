## 2. Content-Plan nach Seitentyp

### Homepage (/)
**Keyword-Fokus:** "Dachdecker München", "Dachdeckerei München"

**Inhalt:**
- Hero mit Notfall-Telefonnummer (prominent)
- Trust-Signale: Meisterbetrieb seit XXXX, XX+ Jahre Erfahrung, XX+ Projekte
- Service-Übersicht (6 Hauptleistungen mit Icons)
- Stadtteil-Quick-Links
- Kundenbewertungen (Google Reviews einbinden)
- FAQ-Sektion (5-8 häufige Fragen)
- CTA-Banner für Kostenloses Angebot

**Technische SEO:**
- LocalBusiness Schema
- FAQPage Schema
- BreadcrumbList Schema

---

### Service-Seiten (z.B. /dachsanierung)
**Keyword-Fokus:** "Dachsanierung München", "Dach sanieren Kosten"

**Content-Struktur:**
1. **Hero:** Hauptüberschrift + Kurztext + CTA
2. **Leistungsübersicht:** Was beinhaltet die Leistung?
3. **Preistransparenz:** Preistabelle oder "ab XX Euro"
4. **Ablauf:** 4-5 Schritte visualisiert
5. **Vorteile:** Warum 089dach?
6. **Referenzprojekt:** 1-2 Beispiele mit Vorher/Nachher
7. **FAQ:** 8-12 spezifische Fragen
8. **Stadtteile:** Interne Links zu lokalen Seiten
9. **CTA:** Kontaktformular + Telefon

**Technische SEO:**
- Service Schema
- FAQPage Schema
- Review Schema (wenn Bewertungen)

---

### Stadtteil-Seiten (z.B. /muenchen/schwabing)
**Keyword-Fokus:** "Dachdecker Schwabing", "Dachsanierung Schwabing"

**Content-Struktur:**
1. **Hero:** "Ihr Dachdecker in [Stadtteil]"
2. **Lokaler Bezug:** Beschreibung des Stadtteils, typische Bausubstanz
3. **Services vor Ort:** Verlinkte Leistungen
4. **Anfahrtszeit:** "In XX Minuten bei Ihnen"
5. **Referenzen:** Projekte in diesem Stadtteil
6. **Notdienst-Hinweis:** 24h erreichbar
7. **Kontakt-CTA:** WhatsApp + Telefon

**Technische SEO:**
- LocalBusiness Schema mit Geo-Koordinaten
- AreaServed erweitern

---

### Ratgeber-Artikel (z.B. /ratgeber/dachsanierung-kosten)
**Keyword-Fokus:** Long-Tail Keywords, informational queries

**Content-Struktur:**
1. **Einleitung:** Problem/Frage aufgreifen
2. **Hauptinhalt:** 1500-2500 Wörter, gut strukturiert
3. **Tabellen/Listen:** Kostenübersichten, Vergleiche
4. **Bilder:** Infografiken, Beispielbilder
5. **Fazit:** Zusammenfassung + CTA
6. **Autor-Box:** Meister XY mit Qualifikation

**Technische SEO:**
- Article Schema
- FAQ Schema (wenn Fragen enthalten)
- HowTo Schema (bei Anleitungen)

```json
  "@context": "https://schema.org",

  "image": "https://089dach.de/logo.png",
  "telephone": "+49-XXX-XXXXXXX",
  "email": "info@089dach.de",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "Musterstraße 1",
    "addressLocality": "München",
    "postalCode": "80XXX",
    "addressCountry": "DE"
  },
  "geo": {
    "@type": "GeoCoordinates",

  "areaServed": [
    {"@type": "City", "name": "München"},
    {"@type": "AdministrativeArea", "name": "Schwabing"},
    {"@type": "AdministrativeArea", "name": "Bogenhausen"}
  ],
  "priceRange": "$$",
  "openingHoursSpecification": {
    "@type": "OpeningHoursSpecification",

    "opens": "07:00",

```

### Core Web Vitals Ziele
- LCP (Largest Contentful Paint): < 2.5s
- FID (First Input Delay): < 100ms
- CLS (Cumulative Layout Shift): < 0.1

### Mobile-First
- Sticky Telefon-Button
- WhatsApp-Integration
- Touch-freundliche CTAs (min. 48px)
- Schnelle Ladezeiten (< 3s auf 3G)
